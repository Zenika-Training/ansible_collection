## Requirements

- Minimum Ansible version: `2.14`

## Default Variables

### ocp_init_api_url

OpenShift API server URL. Used to run `oc login` when the kubeadmin-password file is present.
Defaults to the standard URL pattern produced by the oc_coreos role.

#### Default value

```YAML
ocp_init_api_url: https://api.{{ inventory_hostname }}.{{ oc_coreos_ocp_domain 
  }}:6443
```

### ocp_init_cluster_admins

List of users to grant cluster-admin ClusterRoleBinding.

#### Default value

```YAML
ocp_init_cluster_admins:
  - admin
```

### ocp_init_disable_self_provisioning

When true, removes the self-provisioner ClusterRoleBinding from all authenticated users.
The autoupdate annotation is set to "false" to prevent OpenShift from restoring it.

#### Default value

```YAML
ocp_init_disable_self_provisioning: true
```

### ocp_init_groups

List of OpenShift groups to create. Each entry has a name and a list of users.

#### Default value

```YAML
ocp_init_groups:
  - name: ops
    users:
      - ops
  - name: developers
    users:
      - developer
```

#### Example usage

```YAML
ocp_init_groups:
  - name: ops
    users: [ops]
  - name: developers
    users: [developer]
```

### ocp_init_htpasswd

Content of the htpasswd file used by the HTPasswd identity provider.
Default credentials for training sessions:
admin / redhatocp (bcrypt)
developer / developer (bcrypt)
ops / ops (md5crypt)
To regenerate hashes:
htpasswd -cBb htpasswd admin redhatocp
htpasswd -Bb htpasswd developer developer
htpasswd -Bb htpasswd ops ops

#### Default value

```YAML
ocp_init_htpasswd: |
  admin:$2y$05$kpp9fiMUQDq/YqUSxdiamuYhIIDxG.G1xu2P9zuLTPX1YKxLw6qOK
  developer:$2y$05$UdzflYHTfgFX.fQEvM5vq.EGNOMSSsZb0VB06lWiJLyUAQn/nrUU6
  ops:$apr1$w2nH4KXH$ZsGh4vQi8K5KZOKhrDdgS0
```

### ocp_init_kubeadmin_password_file

Path to the kubeadmin-password file generated by the OpenShift installer.
When present, the role runs `oc login` automatically to obtain fresh credentials.

#### Default value

```YAML
ocp_init_kubeadmin_password_file: /tmp/openshift/{{ inventory_hostname }}/{{ 
  inventory_hostname }}/auth/kubeadmin-password
```

### ocp_init_kubeconfig

Path to the kubeconfig file written by `oc login`. When the kubeadmin-password file
is present, this file is created or refreshed automatically. Otherwise the role falls
back to ~/.kube/config populated by a prior `oc login`.

#### Default value

```YAML
ocp_init_kubeconfig: /tmp/openshift/{{ inventory_hostname }}/{{ 
  inventory_hostname }}/auth/kubeconfig
```

### ocp_init_validate_certs

Whether to validate TLS certificates when connecting to the cluster API.
Set to false for training clusters that use self-signed certificates.

#### Default value

```YAML
ocp_init_validate_certs: false
```
